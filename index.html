A configured connection class was added to the Service folder

The connection object SalesforceConnection will read the configuration values from web.config, enabling your team to change the endpoint and auth configuration for different scenarios.

Your web.config has new configuration values

<appSettings>
    <...>
    <!-- The Salesforce connected app is named: YourProjectName -->
    <add key="Salesforce:ConsumerKey" value="yourKey" />
    <add key="Salesforce:ConsumerSecret" value="yourSecret" />
    <add key="Salesforce:RedirectUri" value="/ProjectName-SalesforceRedirectHandler.axd" />
    <add key="Salesforce:Domain" value="https://login.salesforce.com" />
</appSettings>

Salesforce Connect App was added

Salesforce requires the configuration of a Connected App allowing your app to make requests of the Salesforce Service. Based on the authentication strategy you've chosen, the Connected App was configured, and the configuration values have been saved to web.config. You may change the Connected App in Salesforce.com.
