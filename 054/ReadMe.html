CTP2 Updates

Welcome to CTP 2, our "road to Dreamforce"

In this update, we've made several improvements based on the early feedback and fixed some known issues:

  • Create/Update no longer using workaround for Anonymous Types. All properties of Model classes now include Createable/Updateable attributes and the 0.5.4 SDK now looks for those attributes to filter them from the REST Create/Update calls
  • DisplayName (Label) - The T4 templates now expose the SOObject to leverage properties like the Label property in Salesforce metadata. Visual Studio features like MVC Scaffolding will generate HTML labels that utilize the labels that were pulled from Salesforce. This begins to round out the Metadata features we've been targeting for the Connected Services experience
  • Click Twice, now Once - The OAuthHander scaffolding has been updated to fix the Click Twice on the Salesforce Contacts link in the walkthrough. We now capture the session state properly, with a corresponding fix in the 0.5.4 Force .NET SDK
  • Connected Services moved to the Service References folder. We received feedback the Services folder was where you placed the code you emitted your services, not for where you consumed your services. This work is partially complete. We simply place the service in a folder named Service References, consistent with the current Add Service Reference experience. If you have one, we use it. If you didn't have a Service References folder in your project, while we currently create the directory, we didn't yet do the work to treat it as a "speshal folder".
    Work Around: You can right click on your project, choose Add Service Reference, than hit cancel. This will promote the Services References folder to a Special Folder.
  • Guidance Documentation (Getting Started)
    • Docs have been updated to the new HTML format
    • Getting Started docs available for re-entrancy
      • Under Service References\Salesforce we have placed an HTML file that contains a redirect to the public hosted content.
    • Launched in your default browser, instead of within Visual Studio as the VS browser didn't support JavaScript to support Copy Code to your Clipboard, nor links to external pages.
      • Note: known issue below
    • We're ready for some feedback here on Getting Started
      • How easily can you read the content?
      • What additional content would you find helpful?
      • Copy the code to the clipboard, paste into VS - is this working for you?

Known Issues in this release:

  • IE Security Warning: The guidance docs now open in your default browser. However, the code to do the redirect is currently using JavaScript, rather than the META REFRESH tag. You can either choose to accept the warning for each project, or change the default IE settings to allow javascript for local content. Not recomended, but if you're giving demos, like a PM on the team, you may want to disable it to avoid the conversation about protected content.
  • SalesforceService.Default - We have been playing with some ideas about simplifying the code for instancing a ForceClient. We tried the Default pattern, but realized it could cause some problems with state being shared across threads. The GettingStarted docs use the recommended pattern for this build.

Other Updates

  • Submit A Bug now working
    Using Connect, you can now submit a bug that flows directly into our TFS database for the team to review and triage. We appreciate the emails, but please log bugs into Connect and refer to them in email so you can track it's progress and we can capture it in your voice
  • Submit an Idea now enabled
    This links out to User Voice, it's currently locked down to just the CTP members, so please put ideas, discussions on ideas here. This is the place for you to grow your community, and be the leaders as we open it to the public this coming November.

Installing the Salesforce Connected Service Provider

  • Install Microsoft.VisualStudio.ConnectedServices.vsix
    This installs the base Connected Services Experience
  • Install Salesforce.VisualStudio.Services.vsix
    This installs the Salesforce Provider
  • Launch Visual Studio 2013
  • Create a new Web Project using the ASP.net Web Application Template
  • Choose the MVC Template
  • Once the project opens, right-click on the Project and choose Add Connected Service
