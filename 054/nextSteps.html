Developing with Salesforce

Depending on the options you've chosen, there may be some mandatory next steps to query Salesforce

Web Server Flow

If you've selected Web Server Flow for your runtime authentication strategy, you'll need to update web.config with the Consumer Secret from the Salesforce Portal.

  • Open web.config and note the Connected App name 
    <appSettings>
    <!-- The Salesforce connected app is named: YourProjectName-Dev -->
    <add key="Salesforce:ConsumerKey" value="yourKey"/>
    ...
  • Log into the Salesforce Portal.
  • Scroll down to view the Connected Apps
  • In the API (enable OAuth Settings) section, find the Consumer Secret and click the link to show the key.

  • Copy the consumer key value from the Salesforce Portal and paste it in your application's web.config file as the value of the Salesforce:ConsumerSecret app setting.
  • For C# projects, remove the build #error reminder in Service References\Salesforce\SalesforceOAuthRedirectHandler.cs
