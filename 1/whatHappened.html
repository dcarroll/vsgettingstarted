Developing with Salesforce

Here is how your project has been updated to use Salesforce

References have been added to your project

NuGet package references

  • developerforce.force

.NET references

  • System.Configuration
  • System.Web
  • System.ComponentModel.DataAnnotations *

* added if you've chosen to Scaffold the Salesforce Objects

A configured connection class was added to the Service folder

A configured service class, SalesforceService.cs/vb was added to the Service References folder. This class will provide a configured connection, reading values from app/web.config, providing a ForceClient to execute Salesforce REST queries.

With the values being read from app/web.config, you can manage the configurations as they change through your deployment environments.

Your app/web.config has new configuration values

Depending on the project type and the authentication strategies you've chosen, some of the following entries have been added.

<appSettings>
    <!-- The Salesforce connected app is named: YourProjectName-Dev -->
    <add key="Salesforce:ConsumerKey" value="yourKey" />
    <add key="Salesforce:ConsumerSecret" value="yourSecret" />
    <add key="Salesforce:RedirectUri" value="/ProjectName-SalesforceRedirectHandler.axd" />
    <add key="Salesforce:Domain" value="https://login.salesforce.com" />
    <add key="Salesforce:Username" value="ToBeProvided" />
    <add key="Salesforce:Password" value="ToBeProvided" />
    <add key="Salesforce:SecurityToken" value="ToBeProvided" />
</appSettings>

A Connected App was created in the Salesforce Portal

Based on the authentication strategy you've chosen, a Connected App was configured enabling your application to be authenticated against the Salesforce REST APIs. The configuration values have been saved to app/web.config. You may change the Connected App in the Salesforce Portal.

To find the name, open app/web.config, and look for the following line:

<appSettings>
    <!-- The Salesforce connected app is named: YourProjectName-Dev -->
</appSettings>
Deployment Tip

When you're ready to deploy your application to another environment, you'll need to create another Connected App with the appropriate settings, particularly the callback url, and update the app/web.config file to the target environment.
